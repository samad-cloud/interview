Extract and generalize the SkillInput pattern from `frontend/app/gen-job/page.tsx` (lines 72-142). The existing pattern uses useState for a local `input` value, adds chips on Enter key, and removes them with an X button.

Props:
```typescript
interface SkillChipInputProps {
  skills: string[];
  onChange: (skills: string[]) => void;
  placeholder?: string;
  disabled?: boolean;
  chipColor?: string; // hex color for chip background. Default: '#6366F1'
}
```

Behavior:
- Local `inputValue` state for the text input
- On Enter key (or comma key): trim the value, skip if empty or duplicate, call `onChange([...skills, trimmedValue])`, clear input
- Each chip renders as a small pill with: text, an X button that calls `onChange(skills.filter(s => s !== chip))`
- Chip styling: inline style `{ background: chipColor + '20', color: chipColor, border: '1px solid ' + chipColor + '40' }` — NOT dynamic Tailwind
- Input styled with `className="bg-transparent border-0 outline-none text-sm text-[#F9FAFB] placeholder-[#6B7280] flex-1 min-w-[120px]"`
- Container: `flex flex-wrap gap-1.5 p-2 rounded-lg border border-[#1E293B] bg-[#0F172A] focus-within:border-[#6366F1] min-h-[40px]`

Disabled state: input is `readOnly`, chips are not removable.

Export named: `export function SkillChipInput(...)`.
